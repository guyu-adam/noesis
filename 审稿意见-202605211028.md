## 一、稿件概要

本文实现了一个 5 处理器 RNN 系统（512 神经元/处理器，2560 总神经元），在 8 种实验模式 × 100 cycles = 800 个独立观测下测试了 CGWT 的核心预测。Φ 从神经状态转移概率矩阵（TPM）近似计算，包含 MI integration、effective information、differentiation 三个加权分量。关键方法学创新包括：Φ_within/Φ_between 分解、attention weight 追踪、ANCOVA + Mann-Whitney 双轨统计、权重敏感性分析、Φ 校准锚点、随机划分偏误验证。

### 2.3 Methods — ⭐⭐⭐⭐⭐ 优秀

**架构描述**（Section 2.1）：方程完整，参数明确（512 神经元, 32-dim 输入, 25 unroll steps, 光谱半径 < 0.9）。五类处理器的 Table 1 和 Table 2 清晰描述了 8 种实验模式的逻辑链。Modes 7-8 标注为"methodological controls added in response to peer review"——必要的学术诚实。

**Φ 近似**（Section 2.3）：四项近似以编号列表呈现，每项包含简化说明和偏误讨论：
- State space reduction：clustering + fixed seed
- No MIP search：100 random partition sampling 已确认偏误一致性
- Histogram-based MI：quantile binning, spatial-ergodicity assumption 标注了参考
- Weighted composite：实际报告了 CV = 0.35-0.51 > 阈值 0.3，但通过跨条件一致性论证可接受

审稿人特别肯定 sensitivity CV 的诚实报告——没有对超过阈值的事实进行遮掩，而是用"all modes occupy a narrow CV range"的论证解释了为什么不影响跨条件排序。

**Φ 分解**（Section 2.4）：Equation 3 定义了 Φ_total = Φ_within + Φ_between。关键警告"Φ_within 和 Φ_after 来自不同算法路径、量纲不同，只能跨模式比较各自分量"直接回应了之前的审稿意见。

**Merge 策略**（Section 2.5）：给出了 attention-weighted merge 的完整公式，并预先声明了其局限性（"depends on processors producing sufficiently differentiated representations"）——为 Findings 4 的分析埋下了伏笔。

**统计分析**（Section 2.8）：双轨策略（MWU + ANCOVA）描述清晰。Bonferroni 校正（α_adjusted = 0.05/9 = 0.0056）已添加到方法描述中。

### 2.4 Results — ⭐⭐⭐⭐⭐ 优秀

Results 的核心改进是按证据强度而非假设编号组织，形成了"四个发现"的叙事阶梯：

**Finding 1（确信，H5b）：Broadcast 是 Φ 的必要条件。**
Table 3 数据完整，H5b 的 d = 7.23 效应量极其巨大。ANCOVA F(7,791) = 24.03, p ≈ 0, η² = 0.17。

**Finding 2（确信，H7b）：处理器差异化是关键前提。**
H7b d = 1.12，对 homogeneous 条件给出了方法学洞察：ΔΦ 更大但 Φ_after 更低，"illustrates why phi_after is the appropriate DV for this hypothesis"——这个论证对读者理解双轨统计策略很有帮助。

**Finding 3（确信，H6）：多处理器架构引入信息瓶颈。**
Single-processor > all multi-processor。Self-broadcast ≈ single (p = 0.544, d = 0.007) 干净地分离了"广播机制"和"分布式架构"的效应。Φ_within/Φ_between 的分解分析解释了机制来源。

**Finding 4（重要约束）：CGWT 核心预测未获支持，但有机制解释。**
所有 H1-H4 不显著。Attention weight 数据（cosine 0.955 ± 0.003, retention 0.008）直接量化了 merge 退化的程度。分析框架到位："not a falsification of CGWT but a discovery about its boundary conditions"。

**Φ calibration** 的讨论诚实且准确——over-clustering artifact (n_clusters=30 > n_history=25) 被正确归因，不影响跨条件比较的结论。

### 2.5 Discussion — ⭐⭐⭐⭐⭐ 优秀

Discussion 是本文的最大亮点。五个小节的逻辑链清晰：

1. **Computational Constraints（Section 4.1）：** 将三个核心发现提升为"任何 GWT-IIT 整合框架必须满足的约束条件"，论证有力。

2. **Coalition Consensus Paradox（Section 4.2）：** 将 null result 转化为理论洞察——"coalition is only meaningful when there is genuine disagreement to resolve"。提出了 adaptive coalition 的概念，与 GWT 的 ignition threshold 和 IIT 的 meta-stability 对接。

3. **IIT-FEP Bridge（Section 4.3）：** 审稿人认为这两节是本文最具理论深度的段落。Equation 4 将 coalition selection 形式化为 MaxCal 优化问题——这正是之前审稿意见中建议的"将 coalition 选择从 scoring 升级为 optimization"。第一项（cosine similarity scoring）已在当前实现中操作化；第二项（prediction error penalty）的缺失解释了为什么 coalition 退化为均值。与 Luppi et al. (2024) PID 框架的对接清晰：competitive 保留 unique information，mean-merge 摧毁 synergy。

4. **Limitations（Section 4.4）：** 6 项限制都具体且有建设性——exact Φ validation 缺失、calibration over-clustering、处理器特化不足、单一架构、硬件限制、无行为任务。

5. **Future Work（Section 4.5）：** 四个方向有先后顺序，adaptive coalition 排在第一位（正确）。

### 2.6 Conclusions — ⭐⭐⭐⭐⭐ 优秀

三句话概括了发现、CGWT null result 的诊断价值、以及三个待解决问题的清晰陈述。无浮夸声称。"empirically grounded starting conditions for each of these problems"——精准。

## 四、最终评价

### 稿件优势

1. **学术诚实性**：在 conscious computing 这个充斥着过度声称的领域，本文对 null result 的处理堪称范例——既没有遮掩不支持的假设，也没有夸大次要发现，而是将 null result 理论化为对未来工作的建设性约束。

2. **方法透明度**：四个 Φ 近似全部以编号列表呈现，每个附带了偏误讨论和缓解策略。sensitivity CV 超过自设阈值的事实被诚实地报告而非选择性省略。Calibration artifact 被明确归因。

3. **统计适当性**：双轨策略（MWU + ANCOVA）解决了 ΔΦ 信噪比不足的问题。每个假设标注了 DV 类型和选择理由。Cohen's d 和效力分析提供了效应量解释。

4. **理论深度**：Discussion Sections 4.2-4.3 将该工作与 Luppi et al. (2024) 的 synergy-redundancy 框架和 Kearney (2026) 的 MaxCal/FEP 桥接建立了精确的形式化对应。这不是简单的"我们的结果与 X 一致"，而是将对方的框架用作解释自己数据的概念工具。

5. **可复现性**：commit hash、配置文件、完整依赖列表、raw JSONL 数据归档。

### 稿件弱点

没有致命弱点。两份微小的自引和 calibration 措辞建议见第三节。
