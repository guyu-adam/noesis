## 一、分支定位

`noesis-llm` 是 Noesis 的 LLM agent 版本。不同于 `main` 分支（RNN agent + 因果 Φ → 意识科学），本分支的研究问题是：

> **在多 agent 语言系统中，coalition consensus broadcast 是否比 winner-take-all broadcast 产生更好的信息整合？**

这是一个 **multi-agent coordination** 问题，不是意识科学问题。论文叙事应从 "consciousness theory validation" 降级为 "consciousness-inspired multi-agent broadcast comparison"。

---

## 二、与 main 分支的关键区别

| | noesis-llm (本分支) | main |

| Agent | Ollama qwen3:4b LLM | Small RNN (32 neurons) |

| 目标期刊 | AAMAS / JAIR / EMNLP | Entropy / PLOS Comp Bio |

## 三、致命问题

### 1. Φ 公式不统一

`iit.py:phi_proxy()` 和 `iit.py:phi_collaborative()` 使用不同公式。所有实验模式必须使用同一 Φ 公式。

### 2. 实验数据是占位符

论文 `cgwt_paper.tex` 中的表格是占位数据（U 值 = 576.0 不可能）。需要跑真实实验。

### 3. H4 零结果被轻描淡写

竞争 vs 随机无显著差异（p=0.481）。这本身是重要发现——说明 GWT 的 winner-take-all 在 LLM agent 中没有信息整合优势。
