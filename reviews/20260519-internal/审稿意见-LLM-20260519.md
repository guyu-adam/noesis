## 二、可行性评估

### 2.1 本工作的核心贡献

| 维度 | 内容 |
|------|------|
| **研究问题** | 在多 agent LLM 系统中，coalition consensus broadcast 是否比 winner-take-all 产生更好的信息整合？ |
| **方法** | 4 个角色特化的 LLM agent + attention controller + world model consensus scoring + 5 模式对比 |
| **发现** | coalition > competitive > random > no_broadcast（但 competitive vs random 不显著） |
| **差异化** | 5 模式 head-to-head 对比实验范式 + world model 驱动的 coalition selection |

### 2.2 JAAMAS 审稿人视角的优势

| JAAMAS 关注点 | 本工作对应 |
|---------------|-----------|
| Agent coordination mechanism | Coalition consensus broadcast — 一种新的协调机制 |
| Multi-agent communication | Global workspace broadcast 作为 communication backbone |
| Comparative evaluation | 5 模式 head-to-head (competitive/random/none/single/collaborative) |
| LLM-based agents | 4 个 qwen3:4b agent 角色特化 |
| Reproducibility | 代码开源，API 可调 |

### 2.3 JAAMAS 审稿人视角的弱点

| JAAMAS 关注点 | 风险 | 严重度 |
|---------------|------|--------|
| **形式化/理论深度** | 缺少博弈论分析或数学证明，纯实验驱动 | 高 — JAAMAS 偏理论 |
| **实验规模** | 每模式 24 观测，1 个模型，样本太小 | 高 |
| **baseline 不足** | 只和自己比，没有和现有 coordination 方法比 | 中 |
| **novelty 表述** | "consciousness-inspired" 叙事对 JAAMAS 读者是噪音 | 中 |

### 2.4 与 JAAMAS 已有工作的重叠度

搜索 JAAMAS 近年发表的 LLM multi-agent 工作：

- 大量 LLM agent 博弈/协商/辩论 的文章（2023-2025 爆发）
- 有一批 "multi-agent debate improves reasoning" 的工作
- 有一批 "agent communication topology" 的工作
- **未找到** broadcast-based coordination 的 5 模式对比实验
- **未找到** GWT-inspired coalition formation 的工作

结论：**novelty 足够，但需要把叙事从 "意识科学" 转化为 "协调机制设计和评估"。**

## 四、致命问题（投 JAAMAS 前必须修）

### 1. Φ 公式不统一 — 最严重

`iit.py` 中 `phi_proxy()` 和 `phi_collaborative()` 使用不同的计算公式和权重。审稿人一旦对照代码，直接拒稿。

**修复**：统一为一个 Φ 函数，非 coalition 模式下 coalition 相关项自然为 0。

### 2. 实验数据是占位符

论文表格中的 U 值 = 576.0（理论上限），多个比较完全相同，统计上不可能。

**修复**：跑真实实验，接受数据可能不完美（比如 competitive vs random 真的不显著——这反而是诚实的好结果）。

### 3. "意识"叙事对 JAAMAS 不通

JAAMAS 审稿人关心的是 coordination mechanism，不是 consciousness。当前论文的 GWT/IIT 框架叙事会让 multi-agent 审稿人困惑。

**修复**：叙事从 "GWT + IIT 意识理论验证" → "broadcast-based multi-agent coordination: 5-mode comparison"。GWT 和 IIT 降级为 "inspired by" 放在 related work 和 discussion 里提一句即可。

---

## 五、严重问题（强烈建议修）

### 4. 缺少形式化/理论分析

JAAMAS 偏理论。纯实验驱动的文章如果没有形式化支撑，很容易被要求 "add theoretical analysis"。

**修复（选一个）**：
- 方案 A（轻量）：加一个 formal definition section，定义 broadcast function、coalition selection function、information integration metric，用数学语言描述 5 种模式的差异
- 方案 B（深入）：分析 coalition consensus 在什么条件下等价于某个已知的博弈均衡概念，如果有条件可以做

### 5. 样本量不足

- 每模式 24 观测，JAAMAS 审稿人会要求 power analysis
- 只测了一个 LLM (qwen3:4b)

**修复**：
- stimuli 扩展到 20+，cycles per stimulus 至少 5
- 至少加一个对比模型（llama3.2:3b 或 mistral:7b）

### 6. 缺少消融实验

无法判断 Φ 提升来自哪个组件。

**修复**：
- 关掉 world model → 看 coalition selection 退化成什么
- coalition size 从 2 变到 3
- 关掉 consensus threshold → 等效于什么

### 7. 权重硬编码无论证

**修复**：权重扫描（sensitivity analysis），展示排序结论对权重不敏感。

### 8. 缺少与现有 coordination 方法的对比

JAAMAS 审稿人会问：你的 coalition consensus 比 token-passing / blackboard / contract net 好在哪？

**修复**：至少实现 1-2 个简单的 baseline coordination 机制做对比（如 round-robin broadcast、random speaker selection）。

## 八、当前优势（别丢了）

- **5 模式 head-to-head 对比实验范式**在 multi-agent LLM 文献中未见先例

- 有可运行代码 → 实验可复现
- Coalition > competition 的结论如果真实成立，对 multi-agent 系统设计有实际指导意义
