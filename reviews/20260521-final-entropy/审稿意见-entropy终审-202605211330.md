## 一、科学质量

### 1.1 实验设计

9 种实验模式的逻辑清晰且层层递进：

- **Modes 1-3**（competitive, random, no-broadcast）构成广播效应的基线三角
- **Modes 4-6**（collaborative, hybrid, adaptive）构成 CGWT 的核心检验 + 自适应变体
- **Modes 7-9**（single_processor, single_self_broadcast, homogeneous_competitive）构成方法学控制，隔离因果因素

新增的 adaptive 模式（Mode 9）实现了一个 differentiation gate：当 attention weight 标准差低于阈值（τ = 0.02）时跳过 coalition 改用 competitive selection。实验结果是 attention_weight_std = 0.0007 << 0.02，100% 触发 competitive fallback。这个"失败"本身是有信息量的——它证明了门控机制在概念上有效，同时精确量化了当前处理器特化策略的不足。

每模式 100 个独立观测、900 总观测周期、GPU 加速、固定种子可复现。实验规模对一篇计算研究论文而言足够。

### 1.2 Φ 近似的透明度

四个近似（state space reduction, no MIP search, histogram-based MI, weighted composite）逐条列出，每条附带偏误方向讨论和缓解策略。Sensitivity CV 超过自设阈值（0.35-0.51 > 0.3）的事实未遮掩，而是用跨条件 CV 窄范围论证可接受性。Calibration 的 over-clustering artifact 被明确标注。这种透明度在意识计算领域的论文中不常见，值得肯定。

一个需要诚实面对的事实：**Φ_approx 与 exact Φ 的秩相关验证仍未完成。** 稿件在 Methods 和 Limitations 中两次声明此为 future work。这对一篇声明"仅比较排序"的论文而言是可接受的，但读者和审稿人须知这是本工作最根本的方法学边界——如果未来验证发现秩相关 < 0.7，当前所有基于排序的结论需要重新检视。

### 1.3 统计分析

双轨策略（Mann-Whitney U + ANCOVA）适当。每个假设标注了因变量类型（phi_after 或 phi_delta）和选择理由。Bonferroni 校正（α = 0.05/9 = 0.0056）已声明。Cohen's d 和效力分析为效应量提供了解释。方法学上无问题。

### 1.4 科学质量评定：良好

## 二、核心发现的可信度

### Finding 1: Broadcast 必要性 — 可信度极高

broadcast vs no-broadcast: d = 7.43, p ≈ 0。这是压倒性的效应量。即使 Φ_approx 存在系统偏倚，12-17 倍的差异不可能纯粹由方法学 artifact 产生。ANCOVA（F(8,890)=21.12, η²=0.16）独立确认。**这条发现是稳的。**

### Finding 2: 处理器分化前提 — 可信度高

differentiated vs homogeneous: d = 1.24, p ≈ 0。大效应量 + 有意义的控制条件设计。homogeneous 的 complexity 指标（0.113）显著低于 differentiated 的对应值（0.204-0.212），提供了独立的建构效度支持。

新增的 **processor pairwise similarity** 数据（10 对处理器，pairwise cosine -0.028 到 +0.231，均值 ~0.037）首次证实了 W_rec 结构差异化产生了不同的动力学状态——这是 Finding 2 的机制基础。**这条发现是稳的。**

### Finding 3: 多处理器信息瓶颈 — 可信度中高

single_processor (0.860) > competitive (0.830) > collaborative (0.785)。差异的绝对幅度（0.03-0.08）小于 Finding 1 和 2 的效应量，但在 100 cycles 的样本量下稳定。single_self_broadcast (0.857) ≈ single_processor (0.860) 的控制干净地分离了"广播机制"和"分布式架构"。**方向可信，但效应量大小在 Φ_approx 的偏倚范围内，应谨慎解释绝对差距。**

### Finding 4: Coalition null result — 诊断可信

H1-H4 全部不显著。attention weight 数据（collaborative cosine sim = 0.960, adaptive = 0.979）直接解释原因。但需要注意：**当前稿件的 attention data 与上一稿不同**（上一稿 collaborative cosine = 0.955, attention std = 0.003；这一稿 cosine = 0.960, std = 0.0015）。这不是数据不一致——是因为两稿是不同的实验运行（timestamp 10:19 vs 13:28），且代码有更新（adaptive mode 添加）。两稿数据在核心模式排序上完全一致（single > competitive > hybrid > collaborative > homogeneous > random >> no-broadcast），变异性在合理范围内。**这条发现的可信度在于其机制解释（attention collapse）而非 null 的 p 值。**

### 5.2 建议在 proof 阶段处理

1. **Φ 值的跨运行变异性** — 协同模式的 Φ_after 在两轮实验运行中从 0.740 变为 0.785（~6%）。虽然模式排序完全一致（single > competitive > hybrid > collaborative），且差异在 100 cycles 的 sampling error 范围内，但在论文中说明"values are stable within a ±0.03 band across runs"或类似措辞可以增加读者对数字精度的信心。这不需要重新跑实验。

2. **添加 Φ_after bar plot** — 建议至少添加一张 Figure 1：9 模式的 Φ_after 柱状图，标注 pairwise 显著性。这将显著提升 Results 的可读性。

3. **Calibration 措辞微调** — 当前 calibration 的 footnote 措辞为 "calibration values should be interpreted cautiously." 建议更精确地表述："Calibration values exceed experimental Φ values because n_clusters=30 > n_history=25 creates sparse TPMs that inflate EI. The over-clustering is uniform across reference systems, so the relative ordering of calibration anchors (noise ≈ sine ≈ correlated) is uninformative, but the absolute values should not be compared to experimental Φ."

4. **Adaptive mode 的 DIFF_GATE 阈值选择** — τ=0.02 的理由未在论文中说明。建议在 Methods 中加一句话说明阈值的选择依据（如"chosen to be approximately one order of magnitude above the observed attention weight noise floor"或类似）。

## 六、最终评价

### 优势

1. **约束发现范式**：在意识计算这个充斥过度声称的领域，本文选择了一条更谦逊但也更可持续的路线——不声称"证明"了某个理论，而是发现"约束条件"。三个计算约束（broadcast necessity、differentiation prerequisite、distribution bottleneck）构成了一组互补互限的证据。

2. **方法透明度**：四个 Φ 近似逐一披露，CV 超过阈值的事实未遮掩，calibration artifact 被归因。这在计算方法学论文中是好的标准。

3. **null result 的转化**：H1-H4 的不显著没有被掩埋，而是通过 attention weight + pairwise similarity 的双轨诊断转化为建设性的约束条件。"Coalition consensus requires non-trivial disagreement"这个 formulation 本身就是发现。

4. **MaxCal 操作化**：Discussion Eq.4 是 Kearney (2026) 数学桥接的首次可计算操作化。Adaptive mode 是实现这一操作化的第一步（虽不完整，但方向正确）。

5. **可复现性**：commit hash + config json + raw jsonl + MIT license。

### 弱点

1. **Φ_approx 的秩验证缺失**：这是最根本的方法学边界，稿件已诚实声明
2. **缺少图表**：纯表格呈现，信息密度高但可读性受影响
3. **单架构（tanh RNN）**：鲁棒性未跨架构验证
4. **Collaborative Φ 的跨运行变异性**：minor，但建议在论文中说明

### 贡献判断

本文为一篇合格的计算约束研究。它在 Ferrante et al. (2025) 的对抗性合作范式和 Kearney (2026) 的 MaxCal 桥接之间建立了一个计算方法学节点——既不做对抗也不做纯数学，而是从计算实验中发现约束条件。这种"第三种范式"的位置在当前文献中明确存在且未饱和。
